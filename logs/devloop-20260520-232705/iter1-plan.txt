## How it works

### Input validation (`dependency-graph.ts:5-8`)
- `task_id?: string` — optional, defaults to latest root via `getLatestRoot(db)`
- `format?: 'mermaid' | 'ascii' | 'json'` — defaults to `'mermaid'`

### Tree walking (`dependency-graph.ts:166-171`)
- SQLite recursive CTE starting at root ID, following `parent_id` down the tree
- Fetches all rows in the subtree ordered by `created_at ASC`

### Graph building (`dependency-graph.ts:29-66`)
- `buildGraph()`: produces `{ root_id, nodes[], edges[] }`
- Computes depth via BFS recursion from root following `parent_id`
- Creates `parent_child` edges for every `parent_id` link
- Creates `depends_on` edges for every entry in the `depends_on` JSON array
- Nodes that a task depends on but are **outside** the subtree are silently dropped (line 59 guard)

### Output formatters
- **Mermaid** (`buildMermaid`, lines 72-89): `graph TD` with `T_id["title (status)"]` nodes, `-->` for parent-child, `-.->` for depends_on
- **ASCII** (`buildAscii`, lines 91-142): Unicode box-drawing tree with `[status]` badges and `(depends_on: ...)` annotations
- **JSON** (`buildJson`, lines 144-150): Structured `{ root_id, nodes, edges }`

### Edge cases handled
| Edge case | How handled |
|---|---|
| No task_id provided | Falls back to `getLatestRoot(db)` (returns latest root by `created_at DESC`) |
| No root exists | Returns `{ error: 'No root task found...' }` |
| Task ID not found | Returns `{ error: 'Task not found: ...' }` |
| Empty subtree | Returns `{ error: 'No tasks found in the tree.' }` |
| depends_on references external tasks | Silently dropped (line 59 `taskMap.has(depId)` guard) |
| UUID hyphens in Mermaid IDs | `sanitizeMermaidId` replaces non-alphanum with `_` |
| Cyclic parent_id chains | SQLite CTE detects cycles and stops automatically |

### Edge cases NOT handled (minor)
| Issue | Severity |
|---|---|
| Double-quote `"` in task title breaks Mermaid syntax | Medium — should escape `"` → `#quot;` in label |
| depends_on edges to tasks outside the subtree are invisible | Low — could add as gray nodes with a note, but adds complexity |
| Very large trees → deep recursion in `buildAscii` | Low — UI trees rarely exceed recursion limits |
