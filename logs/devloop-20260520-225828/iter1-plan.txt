### Edge-Case Analysis & Verified Behavior

| Edge Case | Handled? | How |
|---|---|---|
| Invalid/missing file | ✅ | try/catch around read + JSON.parse → returns error |
| Malformed JSON | ✅ | Same catch block |
| Missing root `id` | ✅ | Explicit check returns error |
| Missing root `title` | ✅ | Explicit check returns error |
| Empty children array | ✅ | `node.children \|\| []` |
| Optional fields absent | ✅ | `?? null` / `?? default` on every optional field |
| `depends_on` as non-array | ✅ | `Array.isArray` check → defaults to `[]` |
| `depends_on` ID outside tree | ✅ | `idMap.get()` returns undefined → filtered out |
| Circular parent_id | ✅ | JSON.parse would fail on circular refs |
| Duplicate UUID (v4 collision) | ⚠️ | Extremely unlikely; no explicit collision check |
| Invalid task status string | ⚠️ | Preserved from file; no Zod enum validation |
| Priority out of 1–5 range | ⚠️ | Accepted as-is if number, default 3 if missing |
| Recursion depth overflow | ⚠️ | No depth limit; very deep nesting could stack-overflow |
| Transaction failure | ✅ | Wrapped in `db.transaction()` for atomicity |
| Input not an object | ✅ | `typeof root !== 'object'` check |
