### What's broken / missing

1. **`model_classify` is unlisted** — The `ListToolsRequestSchema` handler's `tools` array omits `model_classify`, `model_route`, and `model_config`. MCP clients can't discover the tool.

2. **Fallback logic mismatch** — When confidence < 0.2 the FALLBACK_MAP runs, but if that also misses, `best` (possibly 0-score) is returned. The spec requires: **if no match, fallback to `high_logic`**. Currently there's no guarantee of that — a 0-score `best` passes through.

3. **Return shape edge case** — `all_scores` is a computed array of all categories, but if the fallback via `FALLBACK_MAP` triggers, it returns a manual object where `all_scores` still references the computed `scores` — that's actually fine, but the `confidence` could be inconsistent with the fallback path.

#### File 2: `src/model-router.ts`

Fix the fallback to guarantee `high_logic` when no meaningful match is found.

**Replace lines 186–206** (the fallback block) to add an explicit `high_logic` fallback after the FALLBACK_MAP check:

```ts
// Try fallback mapping if confidence is very low
if (confidence < 0.2) {
  for (const [key, cat] of Object.entries(FALLBACK_MAP)) {
    if (text.includes(key.toLowerCase())) {
      const rule = CATEGORY_RULES[cat];
      return {
        category: cat, category_name: rule.name, confidence: 0.5,
        matched_keywords: [key],
        all_scores: scores,
      };
    }
  }

  // Ultimate fallback to high_logic
  const defaultRule = CATEGORY_RULES['high_logic'];
  return {
    category: 'high_logic',
    category_name: defaultRule.name,
    confidence: 0.3,
    matched_keywords: [],
    all_scores: scores.map(s => ({ ...s, score: Math.round(s.score * 100) / 100 })),
  };
}
```
