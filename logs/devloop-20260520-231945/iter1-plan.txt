### Edge Cases Handled
- **Task not found** → recorded in `failures` array with reason `'Task not found'`, continues to next
- **Invalid status transition** → recorded in `failures` with the allowed transitions listed, continues
- **Retry increment** → `failed` or `needs_revision` increments `retry_count`
- **Parent cascade** → if `completed`/`approved` and all siblings done, parent auto-completes; counter returned as `cascaded_parent_count`
- **Empty task_ids** → Zod rejects with `min(1)` validation error
- **Transaction atomicity** → all updates in single `db.transaction()` — each task's success/failure is independent
