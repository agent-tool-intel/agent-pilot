### 2. `src/model-router.ts` — Refactor handler

**Changes:**

A) **Remove inline table creation** (lines 344-353) — now in `db.ts`.

B) **Add helper functions** (after `loadPlanConfig`):

```typescript
function getActivePlan(db: Database.Database): string {
  const row = db.prepare("SELECT value FROM app_config WHERE key = 'active_plan'").get() as { value: string } | undefined;
  return row?.value ?? 'B';
}

function seedPlan(db: Database.Database, plan: string, planConfig: PlanConfig, now: string) {
  const insert = db.prepare(
    'INSERT OR REPLACE INTO model_config (plan, category, primary_model, fallback_model, updated_at) VALUES (?, ?, ?, ?, ?)'
  );
  for (const [cat, cfg] of Object.entries(planConfig.task_models)) {
    insert.run(plan, cat, cfg.primary, cfg.fallback, now);
  }
}

function getAvailablePlans(): Array<{ plan: string; name: string; description: string }> {
  return ['A', 'B', 'C'].map(p => {
    const cfg = loadPlanConfig(p);
    return { plan: p, name: cfg.plan_name, description: cfg.description };
  });
}
```

C) **Refactor `handleModelConfig` actions:**

| Action | Changes |
|---|---|
| `list` | Use `input.plan || getActivePlan(db)`. After fetching rows (or seeding if empty), **append `available_plans`** key from `getAvailablePlans()`. Return `{ active_plan, plan_name, description, categories, available_plans }`. |
| `switch` | `const plan = input.plan` (required). Call `seedPlan(...)` if `plan` not yet seeded. **Set `app_config.active_plan = plan`** via `INSERT OR REPLACE`. Return `{ active_plan: plan, plan_name, message }`. |

### Edge Cases Covered

| Edge Case | Handling |
|---|---|
| First run, no DB | `db.ts` seeds plan B + sets `active_plan = 'B'` |
| `list` before any switch | Uses `active_plan = 'B'` from `app_config` |
| `list` with explicit plan | Shows that plan's config (seeds on demand if empty) |
| `list` without enough context | Fallback to `getActivePlan()` → 'B' |
| `switch` to already-active plan | Still re-seeds and updates `active_plan` (idempotent) |
| `set` on non-seeded category | Reads default fallback from `loadPlanConfig(plan)` |
| `set` without `fallback_model` | Preserves existing row's fallback, or falls back to plan JSON default |
| `set` without `category` or `primary_model` | Returns error via zod validation + manual check |
| `reset` without `plan` | Resets active plan (does not change which plan is active) |
| `reset` with explicit `plan` | Resets that specific plan |
| Plan JSON file missing | `loadPlanConfig` throws, caught by index.ts try/catch → error response |
| `switch` / `set` / `reset` with invalid `category` | Accepted (category is free-text); plan JSON may have no match → `set` still writes, `list` shows it |
